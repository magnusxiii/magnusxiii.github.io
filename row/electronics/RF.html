📖 Tips για κάθε format

Hex: με ή χωρίς spaces. Παράδειγμα: 70 07 FC ή 7007FC

Binary: οποιοιδήποτε χαρακτήρες εκτός 0/1 αγνοούνται. Παράδειγμα: 10101 0011 1100

Pulses: + για pulse (ON), - για gap (OFF), σε microseconds. Παράδειγμα: +312 -980 +940 -320 ή απλά αριθμοί με spaces (πρώτος = pulse).

PDV URL: Επικόλληση URL ολόκληρο. Το hash περιέχει encoded pulses (rtl_433 format).

rtl_433 -A output: Επικόλληση όλο το output από rtl_433 -A. Θα εξαχθούν αυτόματα τα pulses.

URH bits: Από Analysis tab → "Bits" view. Π.χ. 1010 1100 0011 0011.

RfRaw: Sonoff RF Bridge format. AA B0 ... 55.

Flipper: Το RAW_Data: line από .sub αρχείο.

🆕 Multi-burst: Αν το input έχει [Pause: N samples] markers (π.χ. URH export με πολλά πατήματα), το tool τα ξεχωρίζει αυτόματα. Εμφανίζεται dropdown για να διαλέξεις «όλα ενωμένα» ή ένα συγκεκριμένο burst. Χρήσιμο για rolling code (σύγκριση πατημάτων) ή για clone ενός μόνο burst.

🆕 Byte alignment: Αν τα bits δεν είναι πολλαπλάσιο του 8 (συχνό σε raw RF), το tool σε ειδοποιεί. Τα byte-based exports (hex dump, C array) κάνουν pad το τελευταίο byte· τα bit-exact (bits dump, pulses, ATtiny με SIGNAL_BITS) στέλνουν ακριβώς τον σωστό αριθμό bits.
